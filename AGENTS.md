# aespa 小卡代購／切卡下單網站原型

## 架構總覽
- `index.html`：單頁原型，包含前台與後台主要畫面。
- `styles.css`：手機優先視覺設計與響應式排版。
- `checkout.css`：下單提醒、付款期限與黑名單規則樣式。
- `warnings.css`：前台注意事項與警示標語樣式。
- `entry-admin.css`：入口頁、商品管理清單與庫存摺疊樣式。
- `app.js`：假資料、庫存計算、頁籤切換、下單預覽互動。
- `README.md`：執行方式與原型說明。

## 不可更動限制
- aespa 成員固定為 Karina、Giselle、Winter、Ningning。
- Winter 預設自留 2 張。
- Giselle 自留選項需存在，預設 0 張。
- 每個商品總套數最多 5 套。
- 商品品項固定為：柚、吉、冬、寧、柚吉、柚寧、冬吉、冬寧、吉寧、套收。
- 庫存不足、商品非收單中時，前台不可下單。
- 後台新增商品時需自動帶入固定品項，且允許管理員手動刪除不開放品項。
- 品項圖片預覽需限制尺寸，避免大圖在手機或桌機後台撐破版面。
- 顧客下單後需於下單當天 23:59 前完成轉帳。
- 同一 Email 或手機有兩次未付款取消紀錄時，需鎖定不可再下單。
- 手動取消未付款訂單時，庫存需自動回架。
- 顧客姓名需提醒填本名，供後續賣貨便品項辨識使用。

## 模組依賴關係
- `index.html` 載入 `styles.css` 與 `app.js`。
- `app.js` 不依賴外部套件，方便用靜態檔直接預覽。

## CRITICAL 邏輯
- `app.js` 中庫存計算與品項扣庫存規則屬於核心邏輯。
- 修改或刪除 `// CRITICAL:` 標記前，需先確認需求並建議 commit。

## 前台頁面範圍
- 首頁需讓顧客理解非現貨代購／切卡。
- 商品列表需清楚顯示商品狀態。
- 商品詳情需呈現品項、價格與售完狀態。
- 下單區需顯示商品摘要與顧客資料欄位。
- 下單區需顯示付款期限、黑名單規則與本名提醒。
- 回填五碼需要求訂單編號與聯絡資訊。
- 訂單查詢需避免透露非本人訂單資訊。

## 後台頁面範圍
- 後台總覽需顯示待查帳、待到貨、Email 失敗與低庫存。
- 商品管理需支援商品狀態、總套數、自留數量與價格設定。
- 商品管理需支援商品預覽圖、品項名稱與品項圖片。
- 商品管理需支援人民幣成本、手動匯率、台幣成本估算與品項單價。
- 商品管理需支援移除已選品項圖片。
- 庫存管理需顯示總套數、自留、可售、已售與剩餘。
- 訂單列表需支援查帳與狀態判斷。
- Email 紀錄需顯示寄送狀態與失敗資訊。
- 到貨通知需先檢查賣貨便連結與通知名單。

## 設計原則
- 手機優先，桌機加寬後再做多欄。
- 商品圖片與狀態優先於裝飾。
- 表單使用單欄，欄位標籤不可只依賴 placeholder。
- 重要按鈕高度至少 44px。
- 不使用過度可愛或大型電商風格。
- 後台以查帳、庫存、訂單處理效率為優先。

## MVP 邊界
- 不做會員系統。
- 不做購物車。
- 不做線上刷卡。
- 不串接賣貨便 API。
- 不做自動對帳。
- 不做多管理員權限。
- 不做複雜報表。

## 測試提醒
- 核心庫存邏輯未來需補單元測試。
- 下單 API 未來需使用資料庫交易或 RPC 防超賣。
- Email 失敗需能保留紀錄並重新寄送。
